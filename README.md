# Когортный анализ удержания пользователей

## О проекте

Проанализировала удержание пользователей e-commerce сервиса на основе событий (view, cart, purchase) за период сентябрь 2020 — февраль 2021.

**Цель:** найти когорту с аномальным падением удержания и предложить гипотезы для улучшения retention.

Источник (Kaggle): https://www.kaggle.com/datasets/mkechinov/ecommerce-events-history-in-electronics-store/data

## Инструменты

- SQLite (очистка, расчёт когорт и retention)
- Python (pandas, matplotlib, seaborn — визуализация)
- Git / GitHub

## Что сделано

1. Очистила данные: удалила дубликаты, NULL user_id, оставила нужные типы событий
2. Определил когорты через дату первого события пользователя
3. Рассчитала retention rate (1–156 дней)
4. Визуализировала падение удержания (столбчатая диаграмма, 10 дней)
5. Сформулировала гипотезы и рекомендации

### Системная проблема (главный вывод)
Удержание падает со 100% до 3–5% уже на **1-й день** у всех когорт. Это не единичный сбой, а проблема продукта: пользователи не видят ценности в возвращении.

## Гипотезы и рекомендации

1. **Онбординг** — приветственная цепочка email/push в первый день
2. **Улучшить first user experience** — упростить регистрацию, показать ценность сразу
3. **Push-уведомления** — напоминания о незавершённых действиях
4. **A/B-тест** — сравнить удержание с онбордингом и без него

## Файлы проекта

- `retention.sql` — SQL-запрос для расчёта когорт и удержания
- `cohort_retention.csv` — результат SQL (для визуализации)
- `cohort_analysis.ipynb` — Jupyter Notebook с визуализацией и выводами
